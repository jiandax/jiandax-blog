# 概述

- Template
- 定义一个操作中算法的框架，而将一些步骤延迟到子类中，使得子类可以不改变算法的结构即可重定义该算法中的某些特定步骤。



# 应用

- 主要解决：一些方法通用，却在每一个子类都重新写了这一方法
- 有多个子类共有的方法，且逻辑相同
- 重要、复杂的方法可以考虑模板模式



# 优点

- 封装不变部分，扩展可变部分
- 提取公共代码，便于维护
- 行为由父类控制，子类实现。



# 缺点

- 每一个不同的实现都需要一个子类来实现，导致类的数量膨胀



# 注意事项

- 为防止恶意操作，一般模板方法都加上final关键字
